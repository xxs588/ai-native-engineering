# 这篇先回答哪条路才是主线

进入实现阶段之后，AI 最容易犯的错，不是完全不会做，而是特别愿意把一切看起来有用的东西一起铺开。扫描器、缓存、报表、视图、后台页面、异步任务，全都像是在推进项目，但最后经常反而把主资产埋掉了。

所以这一章不先问“还能做什么”，而是先问：

**当前 phase 的官方路径到底是什么，主资产要先通过哪条最小闭环稳定长出来。**

# 这篇会怎么推进

这里会围着一个最小可用闭环来讲清楚：

- 什么算主资产，什么只是围着主资产服务的附属层。
- 当前版本一定要打通哪条入口到落地的官方路径。
- 哪些能力虽然迟早要有，但这一轮提前做只会把实现带偏。
- 为什么 AI Native 项目必须先让主线成立，再让边缘能力往上长。

这一章的重点不是“少做点功能”这种空话，而是教读者怎样判断：**什么是当前该收的主线，什么只是看起来很香的副线。**
