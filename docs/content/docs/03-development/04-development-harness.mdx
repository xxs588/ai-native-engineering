# 这篇先纠正哪个误解

一提开发期治理，很多人第一反应就是再加几条规则、再补几层测试、再多写一点禁止事项。但这还不是 Harness 的主体。

Harness 真正要解决的是另一层问题：

**agent 在开发主干的时候，能不能在一个稳定、可观察、可验证、可回退的 how loop 里持续推进。**

# 这篇会把哪几层接起来

这一章会把开发期最值钱的几层 Harness 连在一起讲，而不是拆成散装工具介绍：

- 执行现场怎么隔离，避免 worktree、环境和运行状态互相污染。
- verifier 怎么最小起步，不让“看起来差不多”混成“已经做对了”。
- 运行时观测怎么提前进入开发，而不是等交付时才想起来补。
- review、checkpoint、rollback 怎么进入默认流程。
- 为什么 guardrails 很重要，但 guardrails 不是 Harness 本体。

如果这一章写对，读者会开始真正理解：开发期治理不是“保险丝”，而是主干开发本身的运行方式。
