# 这一部分开始解决“AI 为什么总像在猜”

很多项目写到这里，代码已经不算少了，但 agent 还是经常给人一种“像在猜”的感觉。原因通常不是模型太笨，而是它看不到运行后的世界，只能靠代码和聊天记录推测外部状态。

# 这一部分要把运行时世界接进控制面

Part 4 解决的是四件事：

- 让 CLI 不只是命令入口，而是日志、指标、DOM、截图和 trace 的观察面。
- 让 Playwright 不只是测试工具，而是运行时 verifier。
- 让数据库、第三方 API、文件系统、IPC 和宿主边界进入同一条交付 loop。
- 让联调、验收、发布、回退和人工升级成为默认流程，而不是每次临场救火。

# 读完这一部分，项目会发生什么变化

如果这一部分真的做对，项目会出现一个很明显的变化：

AI 不再只是“会写代码的回答器”，而是开始具备下面这些能力：

1. 能看见运行中的系统发生了什么。
2. 能把问题定位到具体的运行时信号，而不是凭感觉猜。
3. 能在跨端宿主和外部系统边界上稳定落代码。
4. 能参与交付闭环，在失败时停住、回退、再继续。
