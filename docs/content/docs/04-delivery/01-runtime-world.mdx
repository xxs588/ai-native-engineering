# 这篇先把什么从“黑盒”拉出来

代码能看见，不等于系统能看见。很多交付失败并不是因为 AI 不会写，而是因为它看不到程序跑起来之后发生了什么：日志看不见、页面状态看不见、DOM 看不见、指标看不见，于是最后只能对着源码猜运行时。

CLI 在这里真正值钱的地方，也就不是“能跑命令”，而是它最容易成为 agent 进入运行时世界的入口。

# 这篇会把哪条观察链讲清楚

这一章会把这些原本分散的东西收成同一条 loop：

- 命令执行和应用实例怎么关联起来。
- 日志、指标、DOM、截图和 trace 怎么变成 agent 可读信号。
- 为什么没有可读运行时，很多所谓 debug 其实只是更高级的猜。
- 为什么 Harness 的价值，经常不是让 agent 多做一步，而是让 agent 少瞎猜一步。

这一章收完之后，终端就不再只是“跑脚本的地方”，而是整条运行时观察链的第一入口。
