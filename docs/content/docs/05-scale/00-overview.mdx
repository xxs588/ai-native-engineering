# 这一部分不再问“能不能做完”，而是问“能不能一直跑”

真正难的后半场，通常不是“这一轮能不能交付”，而是项目开始连续跑、并行跑、交接跑之后，系统会不会越来越贵、越来越乱、越来越靠人肉兜底。

# 这一部分要把系统推到自运行视角

这里会连续回答五个问题：

- 为什么真正贵的不是 token，而是整条交付链的低质量吞吐。
- 什么必须写回 repo 成为 system of record，什么不能继续留在旧线程里污染后续判断。
- MultiAgent / Agent Team 什么时候真的值钱，什么时候只是在并行复制混乱。
- Harness 怎么从“能跑”继续长成“会自愈、会清理、会因为失败变稳”。
- 当执行越来越自动化之后，人类真正该退到系统的哪个位置。

# 读完这一部分，应该形成什么新视角

如果 Part 5 只留下一个判断，我希望是：

**AI Native 的终局不是“有更多 agent”，而是终于把项目做成了一套能长期运转、能持续纠偏、能把经验沉淀回系统的工程机器。**
